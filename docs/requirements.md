# ちいさな肯定 MVP 要件定義

## 1. ドキュメント情報
- 作成日: 2026-02-08
- 対象: 精神障害者（うつ病・双極性障害など）の自己肯定感向上支援アプリ
- スコープ: 2週間で実装するMVP

## 2. 背景・目的
- 調子が悪い日に「何もできなかった」と感じやすく、自己否定が強まる課題がある。
- 小さな達成を可視化し、自己肯定感を積み上げる習慣形成を目的とする。
- 医療行為・診断は行わず、日常のセルフケア支援に限定する。

## 3. プロダクトゴール
- 30秒で記録できる導線を提供する。
- 「できたこと」を日次で可視化し、自己否定の頻度を下げる。
- しんどい日でも最低限の行動（非常モード）を完了できる。

## 4. 対象ユーザー
- 主対象: うつ病・双極性障害等で日々の気分変動があるユーザー
- 副対象: 回復期で習慣化を始めたいユーザー

## 5. MVP機能要件
1. オンボーディング（3画面）
- 安心感/自己受容/行動開始を促す文言表示
- 利用規約・プライバシーポリシー同意チェック

2. ホーム（今日の1分）
- 気分記録、できたこと記録、非常モードへの導線
- 当日の肯定メッセージ表示

3. 気分チェック
- 5段階の気分入力（必須）
- 補助トグル（低め/高め、任意）
- ひとことメモ（任意、80文字以内）

4. できたこと記録
- クイックタグ選択（複数可）
- 自由入力（任意、100文字以内）
- 保存後に肯定メッセージ表示

5. 積み上げカレンダー
- 週/月表示切替
- 記録有無、気分傾向の視覚表示
- 日付タップで当日記録の詳細表示

6. 非常モード
- 低負荷タスク3つのチェック
- 入力必須なし
- 「これで十分」メッセージ表示

7. 通知
- 1日1回のやさしいリマインド通知（任意ON/OFF）

## 6. 非機能要件
1. 可用性
- オフラインで主要機能を利用可能（ローカル保存）

2. 性能
- ホーム初回表示 1.5秒以内（中位端末想定）
- 記録保存処理 300ms以内（ローカルDB）

3. セキュリティ/プライバシー
- 体調関連データは端末内保存（MVP）
- 外部送信なし（分析SDK未導入の場合）

4. UX/アクセシビリティ
- 最小フォント16px、主要ボタン高48px以上
- 否定的・命令的文言を禁止
- コントラスト比はWCAG AAを目標

## 7. 成功指標（KPI）
- 初回3分以内の記録完了率 >= 70%
- 7日継続率 >= 30%
- 1日1回以上記録ユーザー率 >= 40%

## 8. 制約・前提
- 2週間MVPのため、高度なAI生成や医療連携は対象外
- 認証機能・クラウド同期はMVP対象外
- 多言語は `ja` / `en` のみ対応

## 9. MVP対象外（次フェーズ）
- 医療機関連携
- 高度な分析ダッシュボード
- アカウント管理・クラウドバックアップ
- パーソナライズ推薦エンジン

## 10. リスクと対策
1. 継続率が伸びない
- 対策: 入力項目を最小化、通知頻度固定（1日1回）

2. 罪悪感を誘発するUI
- 対策: Writer/Designerレビューで禁止表現チェック

3. 計測不足で改善不能
- 対策: Data主導でイベント定義を先に固定

## 11. 受け入れ基準
1. 5画面の主要導線がクラッシュなく動作する
2. 気分・できたこと・非常モードの記録が当日中に反映される
3. カレンダーで過去7日/30日の記録状況を確認できる
4. 同意未チェック時はオンボーディング完了不可
